Objective

Write a program to implement and compare SHA-1, SHA-256, and SHA-512. Demonstrate how even a small change in input causes a large change in the hash output (the avalanche effect). Document observations and explain the importance of hashing in digital signatures.

01 Theoretical Background

A Secure Hash Algorithm (SHA) is a one-way cryptographic function that takes an arbitrary-length input and produces a fixed-length digest. The digest is deterministic, pre-image resistant, and collision resistant — making it impossible to reverse-engineer the original input from the hash.

The Avalanche Effect is the property whereby a tiny change in the input (even flipping one bit) causes approximately 50% of the output bits to change. This ensures that similar messages produce completely different hashes, which is critical for security.

import hashlib


def compute_hashes(message: str) -> dict:
    """Compute SHA-1, SHA-256, and SHA-512 for a given message."""
    encoded = message.encode('utf-8')
    return {
        "SHA-1":   hashlib.sha1(encoded).hexdigest(),
        "SHA-256": hashlib.sha256(encoded).hexdigest(),
        "SHA-512": hashlib.sha512(encoded).hexdigest(),
    }


def count_bit_differences(hash1: str, hash2: str) -> tuple:
    """Compare two hex hashes and count differing bits."""
    bits1 = bin(int(hash1, 16))[2:].zfill(len(hash1) * 4)
    bits2 = bin(int(hash2, 16))[2:].zfill(len(hash2) * 4)
    diff_bits = sum(b1 != b2 for b1, b2 in zip(bits1, bits2))
    total_bits = len(bits1)
    return diff_bits, total_bits


def display_hashes(label: str, hashes: dict):
    print(f"\n  [{label}]")
    for algo, digest in hashes.items():
        print(f"  {algo:8}: {digest}")


def avalanche_analysis(msg1: str, msg2: str):
    h1 = compute_hashes(msg1)
    h2 = compute_hashes(msg2)
    print("\n  AVALANCHE EFFECT ANALYSIS")
    print(f"  Original : '{msg1}'")
    print(f"  Modified : '{msg2}'")
    for algo in h1:
        diff, total = count_bit_differences(h1[algo], h2[algo])
        pct = (diff / total) * 100
        print(f"\n  {algo}:")
        print(f"    Bits changed : {diff} / {total} ({pct:.1f}%)")

All three algorithms exhibit approximately 50% bit change — the ideal avalanche effect. SHA-256 and SHA-512 are preferred for modern applications due to their resistance to collision attacks. SHA-1, while showing the avalanche effect, has known collision vulnerabilities and should not be used in security-critical systems.

05 Importance in Digital Signatures

Hashing is a cornerstone of digital signature schemes. Rather than signing an entire (potentially large) message, the signature algorithm signs the message's hash — a compact, fixed-length representation. This provides two key benefits: efficiency (signing a 256-bit hash is much faster than signing megabytes of data) and integrity (if even one byte of the message changes, the hash changes entirely, invalidating the signature).

The avalanche effect ensures that an attacker cannot make subtle modifications to a signed document while preserving a valid signature. Combined with the one-way nature of hash functions, this makes SHA-256 and SHA-512 essential tools in secure communication systems.
