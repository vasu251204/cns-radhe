01 Theoretical Background

A Message Authentication Code (MAC) is a short piece of information used to authenticate a message. It is generated by combining a message with a shared secret key through a cryptographic algorithm. The receiver, who also possesses the secret key, can verify that the message was not tampered with in transit.

HMAC (Hash-based Message Authentication Code) is the most widely used MAC construction. It applies a hash function (typically SHA-256) to a combination of the message and key in a specific double-hashing structure, defined as: HMAC(K, m) = H((K ⊕ opad) || H((K ⊕ ipad) || m)).

02 Sender Script

Save as sender.py. This script generates the HMAC for a message and outputs the message along with its authentication code.

sender.py
# ============================================================
#  ALA 3 — Sender: Generates HMAC for a message
#  Student: Dhruv Makwana  |  GMIU  |  2025-26
# ============================================================

import hmac
import hashlib
import json


# ─── Shared Secret Key (known to both sender and receiver) ─
SECRET_KEY = b"GMIU_Secret_Key_2025"


def generate_mac(message: str, key: bytes) -> str:
    """Generate HMAC-SHA256 for a given message and key."""
    mac = hmac.new(key, message.encode('utf-8'), hashlib.sha256)
    return mac.hexdigest()


def create_authenticated_packet(message: str) -> dict:
    """Create a data packet with the message and its MAC."""
    mac_value = generate_mac(message, SECRET_KEY)
    return {
        "message": message,
        "mac": mac_value
    }


if __name__ == "__main__":
    print("=" * 60)
    print("   ALA 3 — MAC Sender | Dhruv Makwana")
    print("=" * 60)

    original_message = "Transfer Rs.5000 to Account 987654321"
    packet = create_authenticated_packet(original_message)

    print(f"\n[✔] Message   : {packet['message']}")
    print(f"[✔] HMAC-SHA256: {packet['mac']}")
    print(f"\n[→] Packet transmitted to receiver...")

    # Save packet to simulate transmission
    with open("packet.json", "w") as f:
        json.dump(packet, f)

    print("[✔] Packet saved to packet.json")
    print("=" * 60)
03 Receiver Script

Save as receiver.py. This script reads the transmitted packet, recomputes the HMAC, and verifies whether the message is authentic and unmodified.

receiver.py
# ============================================================
#  ALA 3 — Receiver: Verifies HMAC of received message
#  Student: Dhruv Makwana  |  GMIU  |  2025-26
# ============================================================

import hmac
import hashlib
import json


# ─── Shared Secret Key (same as sender) ───────────────────
SECRET_KEY = b"GMIU_Secret_Key_2025"


def compute_mac(message: str, key: bytes) -> str:
    """Recompute HMAC-SHA256 for verification."""
    mac = hmac.new(key, message.encode('utf-8'), hashlib.sha256)
    return mac.hexdigest()


def verify_packet(packet: dict) -> bool:
    """Verify the received message using HMAC comparison."""
    received_msg = packet["message"]
    received_mac = packet["mac"]
    expected_mac = compute_mac(received_msg, SECRET_KEY)
    # Use hmac.compare_digest for timing-safe comparison
    return hmac.compare_digest(received_mac, expected_mac)


def test_tampered(packet: dict):
    """Test what happens when an attacker modifies the message."""
    tampered_packet = {
        "message": packet["message"].replace("987654321", "111111111"),
        "mac": packet["mac"]  # original MAC, not updated
    }
    result = verify_packet(tampered_packet)
    return tampered_packet, result


if __name__ == "__main__":
    print("=" * 60)
    print("   ALA 3 — MAC Receiver | Dhruv Makwana")
    print("=" * 60)

    with open("packet.json", "r") as f:
        packet = json.load(f)

    print(f"\n[←] Packet Received:")
    print(f"    Message : {packet['message']}")
    print(f"    MAC     : {packet['mac']}")

    # Verify original
    is_valid = verify_packet(packet)
    print(f"\n[✔] MAC Verification (original): {'VALID ✔' if is_valid else 'INVALID ✘'}")

    # Verify tampered
    tampered, is_valid_t = test_tampered(packet)
    print(f"\n[!] Tampered Message : {tampered['message']}")
    print(f"[✘] MAC Verification (tampered): {'VALID' if is_valid_t else 'INVALID ✘ — Tampering Detected!'}")

    print("\n[INFO] MAC ensures integrity — even one character change fails verification.")
    print("=" * 60)

04 Test Scenarios
✔ Scenario 1 — Valid
Original message received with correct MAC. HMAC recomputation matches. Message is authentic and unmodified.
✘ Scenario 2 — Tampered Message
Attacker modifies the message (changes account number) but cannot update the MAC without the key. Verification fails immediately.
✘ Scenario 3 — Wrong Key
Receiver uses a different key to compute the MAC. Even with the original message, the recomputed MAC will not match.
✘ Scenario 4 — Forged MAC
Attacker sends a random MAC value with the original message. Without the secret key, producing a valid MAC is computationally infeasible.
05 Analysis & Security Properties
PropertyHow HMAC Provides It
Data IntegrityAny change to the message produces a completely different MAC, making tampering detectable.
AuthenticationOnly parties with the shared secret key can generate or verify a valid MAC.
Replay ProtectionTimestamps or sequence numbers can be included in the message to prevent replay attacks.
Timing Safetyhmac.compare_digest() prevents timing side-channel attacks during comparison.
No Non-repudiationBoth parties share the key; either could generate the MAC. For non-repudiation, digital signatures are needed.
06 Conclusion

This activity successfully demonstrated the design and simulation of an HMAC-based message authentication system in Python. The sender generates a MAC using a shared secret key and SHA-256, while the receiver independently recomputes and compares the MAC to verify authenticity and integrity.

The experiment proved that HMAC is highly effective against message tampering — even changing a single character in the message (the account number) causes verification to fail immediately. HMAC is widely used in real-world applications including API authentication, TLS record authentication, and JWT (JSON Web Token) signing, making this a critical concept in applied cryptography.
