Digital Signature Verifier

Public-key cryptography: key generation, message signing, and verification using RSA and SHA-256.

Student: Dhruv Makwana
Language: Python 3
Topic: Digital Signatures

Objective

Implement a Digital Signature System using Python to understand the working of public-key cryptography. Generate a key pair, sign a message using the private key, and verify the signature using the public key, with SHA-256 hashing to ensure message integrity.

01 Theoretical Background

A digital signature is a cryptographic scheme that provides authenticity, integrity, and non-repudiation for digital messages. It is the digital equivalent of a handwritten signature, but far more secure due to its mathematical foundation.

In RSA-based digital signatures, the signer uses their private key to produce a signature over the hash of a message. Any recipient can then use the corresponding public key to verify the signature — confirming both the origin and integrity of the message.

SHA-256 is used as the hashing algorithm because it produces a fixed 256-bit digest, making the signing process efficient and collision-resistant.

02 Implementation Steps

  • Install Required Library

    The cryptography library provides RSA key generation, hashing, and signing utilities.

  • Generate RSA Key Pair

    Generate a 2048-bit RSA private key and derive the corresponding public key.

  • Sign the Message

    Hash the message with SHA-256 and produce a digital signature using the private key with PSS padding.

  • Verify the Signature

    Use the public key to verify the signature. Any tampering with the message will cause verification to fail.

from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.backends import default_backend
import base64


# ─── Step 1: Generate RSA Key Pair ────────────────────────
def generate_key_pair():
    private_key = rsa.generate_private_key(
        public_exponent=65537,
        key_size=2048,
        backend=default_backend()
    )
    public_key = private_key.public_key()
    return private_key, public_key


# ─── Step 2: Sign a Message ───────────────────────────────
def sign_message(private_key, message: str) -> bytes:
    message_bytes = message.encode('utf-8')
    signature = private_key.sign(
        message_bytes,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        ),
        hashes.SHA256()
    )
    return signature


# ─── Step 3: Verify a Signature ───────────────────────────
def verify_signature(public_key, message: str, signature: bytes) -> bool:
    message_bytes = message.encode('utf-8')
    try:
        public_key.verify(
            signature,
            message_bytes,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()),
                salt_length=padding.PSS.MAX_LENGTH
            ),
            hashes.SHA256()
        )
        return True
    except Exception:
        return False


# ─── Main ─────────────────────────────────────────────────
if __name__ == "__main__":
    message = "Hello, this is Dhruv Makwana. This message is authenticated."

    print("=" * 60)
    print("   ALA 1 — Digital Signature Verifier")
    print("   Student: Dhruv Makwana | GMIU")
    print("=" * 60)

    # Generate keys
    private_key, public_key = generate_key_pair()
    print("\n[✔] RSA 2048-bit key pair generated successfully.")

    # Sign
    signature = sign_message(private_key, message)
    encoded_sig = base64.b64encode(signature).decode()
    print(f"\n[✔] Message signed with private key.")
    print(f"    Signature (Base64): {encoded_sig[:60]}...")

    # Verify — original message
    result = verify_signature(public_key, message, signature)
    print(f"\n[✔] Verification Result (original message): {result}")

    # Verify — tampered message
    tampered = message + " [TAMPERED]"
    result_tampered = verify_signature(public_key, tampered, signature)
    print(f"[✘] Verification Result (tampered message): {result_tampered}")

    print("\n[INFO] Tampering detected! Signature is INVALID for modified message.")
    print("=" * 60)

The experiment conclusively demonstrates that even a single character added to the message (" [TAMPERED]") causes the verification to fail. This is because the SHA-256 hash of the original and modified messages differ entirely, and the signature only matches the hash of the original. This guarantees message integrity.

The use of RSA's private key for signing ensures that only the holder of the private key could have produced the signature, providing authenticity and non-repudiation.

05 Conclusion

This activity successfully demonstrated the implementation of a digital signature system using RSA and SHA-256 in Python. The experiment proved that digital signatures provide three essential security properties: message integrity (any tampering is detected), authentication (only the private key owner can sign), and non-repudiation (the signer cannot deny signing). These properties are fundamental to modern secure communications, e-commerce, and software distribution.
